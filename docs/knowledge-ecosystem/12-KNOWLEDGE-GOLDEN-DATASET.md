# KNW-KE-ARCH-012 — Knowledge Golden Dataset

**Phase:** 3.0C.5  
**Status:** CANONICAL  
**Owner:** architecture-board  
**Version:** 1.0.0

---

## Purpose

The Golden Dataset is the canonical reference corpus for testing, benchmarking, regression, and AI evaluation. It is curated, not generated. It must always pass the linter with zero errors.

---

## Dataset Location

```
knowledge/dataset/
├── golden/           # curated canonical objects (read-only in CI)
│   ├── objects/      # 200 canonical objects across all 33 types
│   ├── relationships/# 500 canonical relationships
│   └── packages/     # mini-package structure
├── regression/       # objects that previously caused bugs (never delete)
├── performance/      # large-volume synthetic set for benchmarking
│   ├── 10k/          # 10,000 objects
│   ├── 100k/         # 100,000 objects
│   └── 1m/           # 1,000,000 objects (generated, not committed)
└── ai-eval/          # objects for AI/LLM evaluation
    ├── completeness/ # objects with missing fields
    ├── quality/      # objects at various quality scores
    └── reasoning/    # objects for reasoning engine evaluation
```

## Golden Dataset Rules

| Rule | Description |
|------|-------------|
| GD-001 | All golden objects must pass `kos lint` with zero errors |
| GD-002 | All golden objects must have `status` ≥ VERIFIED |
| GD-003 | All golden objects must have `quality.overall_score` ≥ 0.75 |
| GD-004 | All golden relationships must reference objects in the golden set |
| GD-005 | Golden dataset is read-only; changes require ADR + Architecture Board |
| GD-006 | Regression objects may be DRAFT (they test failure modes) |
| GD-007 | Performance dataset is generated by `kos dataset generate --size 1m` |
| GD-008 | AI eval objects span the full quality score range [0.0, 1.0] |

## Regression Dataset

Objects that previously caused bugs:

```yaml
# knowledge/dataset/regression/REG-001-duplicate-id.yaml
# Tests: duplicate knowledge_id detection (was bug #42)
knowledge_id: KNW-PLT-MOD-001   # intentional duplicate of golden object
_regression_context:
  bug_id: BUG-042
  description: "Registry allowed duplicate IDs before fingerprint check"
  fixed_in_phase: "3.0D.1"
```

Regression dataset never shrinks. Each regression object documents the bug it covers.

---

## Performance Dataset Generation

The performance dataset is not committed (too large). It is generated on demand:

```bash
kos dataset generate \
  --size 1m \                    # 1,000,000 objects
  --types module,service \       # which types to generate
  --relationships 10m \          # 10,000,000 relationships
  --output knowledge/dataset/performance/1m/
  --seed 42                      # reproducible
```

**Performance dataset properties:**
- Random but valid knowledge_ids
- Realistic name distribution (sampled from golden names)
- Relationships follow real distribution from golden set
- Always passes the linter

## AI Evaluation Dataset

Used for evaluating LLM-based features (semantic search, reasoning, NL→KQL):

```
ai-eval/completeness/   — objects with 0, 1, 2, ... required fields missing
ai-eval/quality/        — objects at quality scores 0.1, 0.3, 0.5, 0.7, 0.9
ai-eval/reasoning/      — 50 objects with known "why/how/impact" answers
                          for regression testing R-1 through R-10
```

Each AI eval object has a `_ground_truth` section:

```yaml
_ground_truth:
  reasoning_why: "This module was created to enforce rate limits..."
  reasoning_impact: ["KNW-PLT-MOD-002", "KNW-ARCH-REQ-001"]
  quality_score: 0.82
  correct_kql: "SELECT objects WHERE type = 'module' AND namespace = 'plt'"
```
