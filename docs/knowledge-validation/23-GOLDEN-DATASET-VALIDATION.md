# KVF-DOC-023 — Golden Dataset Validation

**Phase:** 3.0D.1.5
**Status:** CANONICAL
**Owner:** architecture-board
**Version:** 1.0.0

---

## Purpose

The Golden Dataset is the authoritative ground-truth benchmark for all KVF
evaluations that require human-annotated expected answers. It is built once
per major KOS corpus version and used across all validation documents.

---

## Golden Dataset Structure

```
The Golden Dataset contains:

  GD-1: Query-Answer Pairs
    For each query Q: the expected correct answer A(Q)
    Annotated by domain experts, not generated by AI

  GD-2: Required Object Sets
    For each query Q: the set of required KnowledgeObjects R(Q)
    Used by: knowledge coverage (doc 07), search quality (doc 08)

  GD-3: Reasoning Traces
    For each REASONING query: the step-by-step correct reasoning chain
    Used by: reasoning quality (doc 09)

  GD-4: Baseline Token Counts
    For each query: the naive token count for manual context assembly
    Used by: token efficiency (doc 06)

  GD-5: Hallucination Examples
    100+ labeled hallucination examples (one per sub-type A–F)
    Used by: hallucination measurement (doc 10)

  GD-6: Compression Ground Truth
    For 50 objects: human-verified L1/L2/L3/L4 summaries
    Used by: compression validation (doc 11)
```

---

## Dataset Construction Protocol

```
Step 1: Object Selection
  Select 100 objects from corpus:
    20 from each of 5 packages
    Quality distribution: 20% per quality band
    Mix of CANONICAL, ACTIVE, DEPRECATED

Step 2: Query Generation
  Generate 500 queries covering all 4 question types (doc 09):
    50 questions per object type × 10 types = 500 total
    Distribute across 12 intent types (doc 03)

Step 3: Human Annotation
  Team of 3 domain experts:
    Each expert answers all 500 queries independently
    Annotators also select required object sets R(Q)
    Resolution: majority vote; contested annotations → expert discussion

  Inter-annotator agreement threshold:
    Factual answers: kappa >= 0.80
    Required object sets: Jaccard >= 0.75
    Below threshold: re-annotate or exclude query

Step 4: Baseline Token Count
  For each query: one annotator manually gathers the files they would open
  Tokens counted with the standard tokenizer
  Minimum 3 independent annotators for 50 sampled queries

Step 5: Hallucination Labeling
  Generate AI answers without KOS context for 500 queries
  Label each hallucination with type A–F from doc 10
  Minimum 100 labeled hallucination instances

Step 6: Compression Verification
  For 50 objects: human expert writes the L1/L2/L3 summaries
  These serve as ground truth for compression validation (doc 11)
```

## Golden Dataset Schema

```yaml
golden_dataset:
  version: string                       # tied to KOS corpus version
  created_at: string
  annotator_count: integer
  query_count: integer

  queries:
    - query_id: string
      query_text: string
      intent_type: string
      question_type: string             # QT-ARCH | QT-DEP | QT-IMPACT | QT-LIFECYCLE
      primary_object_id: string
      required_objects: [string]        # R(Q)
      answer:
        text: string                    # ground truth answer
        source_fields: [string]         # which KIL fields the answer comes from
        confidence: AUTHORITATIVE | CONFIDENT | PROBABLE
      naive_token_count: integer | null
      annotator_agreement: float

  hallucination_examples:
    - example_id: string
      query_id: string
      hallucination_type: string        # A | B | C | D | E | F
      hallucinated_claim: string
      correct_value: string
      detection_method: string

  compression_ground_truth:
    - knowledge_id: string
      L1_human: string
      L2_human: string
      L3_human: string
```

## Dataset Validation (meta-validation)

The golden dataset itself must be validated:

```
GDV-1: Coverage
  GDV-1.01: At least 10 queries per intent type (doc 03)
  GDV-1.02: At least 50 queries involving DEPRECATED objects
  GDV-1.03: At least 100 queries where answer requires > 1 object

GDV-2: Quality
  GDV-2.01: Mean annotator agreement >= 0.80 (factual)
  GDV-2.02: Mean Jaccard (required objects) >= 0.75
  GDV-2.03: No query with 0 required objects (every query needs something)

GDV-3: Completeness
  GDV-3.01: All 6 hallucination types have >= 10 labeled examples each
  GDV-3.02: Compression ground truth covers >= 50 objects
  GDV-3.03: Baseline token counts available for >= 100 queries
```

---

## Rules

| Rule | Statement |
|------|-----------|
| KVF-111 | Golden dataset must be human-annotated — AI-generated ground truth is not accepted |
| KVF-112 | Inter-annotator agreement below threshold requires re-annotation — not averaging |
| KVF-113 | Dataset version must be tied to KOS corpus version — using old dataset on new corpus is invalid |
| KVF-114 | Hallucination examples must cover all 6 types — a type with 0 examples is a dataset gap |
| KVF-115 | Dataset construction must use at least 3 independent annotators per query |
